第一章：加载和运行

    

	阻塞加载：简单来说，就是当 JavaScript 文件运行的时候其他事件不能被浏览器处理.JavaScript 运行了多长时间,那么在浏览器空闲下来响应用户操作之前的等待时间就有多长.

    

	管理浏览器中 JavaScript 代码是个棘手的问题，因为代码执行阻塞了其他浏览器处理过程，诸如用户界面绘制。每次遇到script标签，页面必须停下来等待代码下载(如果是外部的)并执行，然后再继续处理页面其他部分。但是，有几种方法可以减少 JavaScript 对性能的影响：
   

 		1.将所有 script 标签放置在页面的底部，紧靠关闭的 body 标签 </body> 的上方.此方法可以保证页面在脚本运行之前完成解析。
    
    
		2.将脚本成组打包。页面的 script 标签越少，页面的加载速度越快，响应也更加迅速。不论外部脚本文件还是内敛代码都是如此。
    
    
		3.为 script 标签添加 defer 属性（只适用于IE和Firefox3.5以上版本）。
    
    
		4.动态创建 script 标签，用它下载并执行代码( 封装loadScript函数 )。
    
    
		5.用 xhr 对象下载代码，并且注入到页面中。
